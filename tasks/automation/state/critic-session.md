## Current Memory

- The weakest release gate is now `UX/UI`: title, pause, Tab, and forge default surfaces still explain the route and doctrine too early, which flattens the small-start-to-transformation curve before combat can sell it.
- Treat any split between the `8-wave shipped slice` and `Wave 9-12` player-facing scaffolding as immediate drift. Hidden late-route logic is tolerable; exposed late-route cadence, proof ladders, and doctrine copy are not.
- Highest-value critique remains one clean player-facing arc: `lean start -> Wave 3 weapon thesis -> Wave 6 visible support install -> Wave 8 mastery lap`.
- Early-growth rule: the opener should feel physically small. Signatures or ladder copy cannot make the run look strategically solved before the first earned mutation.
- Reward/UI rule: default title/forge/pause/Tab surfaces should behave like a fast status board and object-first pick screen. One dominant reward card and one short combat ask beat roadmap ladders, doctrine chips, and multi-slot payoff shells.
- Combat-space rule: encounter configs already know how to open the arena and reduce suffocation on payoff bands; critique should push for that breathing room to serve clearer movement decisions and a stronger mutation payoff, not for more density theater.
- Repeated false solutions: softer wording for the same taxonomy, preserving roadmap cards with less text, adding more Wave 9-12 scaffolding, or inventing new support layers before the current mid-run install becomes instantly desirable.
